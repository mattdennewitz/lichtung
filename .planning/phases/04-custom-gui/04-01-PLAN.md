  <name>Task 1: Create CustomLookAndFeel and PluginEditor with full 5-section layout</name>
  <files>src/ui/CustomLookAndFeel.h, src/ui/CustomLookAndFeel.cpp, src/PluginEditor.h, src/PluginEditor.cpp</files>
  <action>
Create `src/ui/` directory and implement the custom LookAndFeel and editor.

**CustomLookAndFeel (.h/.cpp):**
- Subclass `juce::LookAndFeel_V4`
- Color constants: background `#1a1a2e`, accent teal `#00d4aa`, panel `#222244`, label gray `#cccccc`
- Override `drawRotarySlider()`: dark arc background (panel color, 3px stroke), teal value arc from start angle to current angle, white indicator dot at current position. Reduce bounds by 2px for padding.
- Override `drawLinearSlider()` for `LinearVertical`: 6px wide track centered horizontally, panel color background with rounded corners (3px radius), teal fill from bottom up to slider position.
- Override `drawComboBox()`: panel color background fill with rounded corners, teal outline, white dropdown arrow.
- Override `drawPopupMenuItem()`: dark background, teal highlight for selected/hovered items, white text.
- Override `drawLabel()`: light gray text, no background fill, no outline.
- Set default colours in constructor via `setColour()` for: ComboBox text (white), ComboBox background (panel), PopupMenu background (background color), PopupMenu text (label gray), PopupMenu highlight (accent teal).

**PluginEditor (.h/.cpp):**

Header — declare members in this order (destruction order matters):
1. `PluginProcessor& processorRef`
2. `CustomLookAndFeel customLookAndFeel` (declared BEFORE child components so destroyed AFTER them)
3. All `juce::Slider` members: `coarseTuneSlider`, `fineTuneSlider`, `glideSlider`, `harmonicSliders[8]`, `scanCenterSlider`, `scanWidthSlider`, `spectralTiltSlider`, `linFmDepthSlider`, `expFmDepthSlider`, `fmRateSlider`, `attackSlider`, `releaseSlider`, `masterLevelSlider`
4. `juce::ComboBox outputSelectBox`
5. `juce::Label` members for section headers (pitchLabel, harmonicsLabel, scanLabel, fmLabel, outputLabel) and per-control labels (harmonic numbers, frequency ratios)
6. All `std::unique_ptr<juce::AudioProcessorValueTreeState::SliderAttachment>` — one per slider (declared AFTER sliders)
7. `std::unique_ptr<juce::AudioProcessorValueTreeState::ComboBoxAttachment> outputSelectAttach`

Constructor:
- Call `setLookAndFeel(&customLookAndFeel)` on `this` (propagates to all children)
- Configure resize BEFORE setSize: `setResizable(true, true)`, `setResizeLimits(525, 338, 1050, 675)`, `getConstrainer()->setFixedAspectRatio(700.0 / 450.0)`, then `setSize(700, 450)`
- For each rotary knob slider: `setSliderStyle(RotaryVerticalDrag)`, `setTextBoxStyle(TextBoxBelow, false, 60, 16)`, `setDoubleClickReturnValue(true, defaultValue)`, `addAndMakeVisible()`, then create SliderAttachment with correct parameter ID
- For `coarseTuneSlider`: also call `setNumDecimalPlacesToDisplay(0)` since it's integer semitones
- For each harmonic fader `harmonicSliders[0..7]`: `setSliderStyle(LinearVertical)`, `setTextBoxStyle(TextBoxBelow, false, 40, 16)`, `setDoubleClickReturnValue(true, defaultVal)` where default is 1.0 for index 0 and 0.0 for rest, `addAndMakeVisible()`, then create SliderAttachment to `"harm_" + String(i+1)`
- For outputSelectBox: `addItemList({"Harmonic Mix", "Triangle", "Sawtooth", "Square"}, 1)`, `addAndMakeVisible()`, then create ComboBoxAttachment to `"output_select"`
- Create section header Labels: set text, font (sans-serif, ~14px bold), text colour (label gray), justification centered, `addAndMakeVisible()`
- Create harmonic number labels (1-8) and frequency ratio labels (1x-8x) for above and below faders

Destructor:
- Call `setLookAndFeel(nullptr)` to clear the LookAndFeel pointer before members are destroyed

`paint(Graphics& g)`:
- Fill entire background with `#1a1a2e`
- Draw 5 rounded rectangles (cornerSize ~6px) with `#222244` fill for each section panel area. Compute section areas the same way as `resized()` to keep them aligned.

`resized()` — all proportional from `getLocalBounds()`:
- `auto area = getLocalBounds().reduced(8)`
- Top row: `area.removeFromTop(area.getHeight() * 0.55f)` — split into Pitch (left ~22%) and Harmonics (remaining ~78%)
- Bottom row: remaining area — split into thirds: Scan/Tilt, FM, Output
- Add small gaps (4px) between sections using `.reduced(2)` or `.removeFromLeft/Top(4)`
- Within each section area, `removeFromTop(~24px)` for the section header label
- **Pitch section:** Stack 3 knobs vertically (coarse tune, fine tune, glide) with labels. Each knob ~60x60 area.
- **Harmonics section:** 8 faders evenly spaced horizontally. Each fader gets equal width column. Reserve top for harmonic number label, bottom for frequency ratio label, middle ~65% for the fader itself.
- **Scan/Tilt section:** 3 knobs (scan center, scan width, spectral tilt) arranged in a row or inverted triangle. ~50x50 each.
- **FM section:** 3 knobs (lin FM depth, exp FM depth, FM rate) in a row. ~50x50 each.
- **Output section:** Attack knob, Release knob in a row at top. Below: output select ComboBox (~120x24) and master level knob. Arrange to fit comfortably.

Default values for `setDoubleClickReturnValue` (must match APVTS defaults):
- coarse_tune: 0, fine_tune: 0, glide: 0
- harm_1: 1.0, harm_2..8: 0.0
- scan_center: 0.5, scan_width: 1.0, spectral_tilt: 0.0
- lin_fm_depth: 0.0, exp_fm_depth: 0.0, fm_rate: 1.0
- attack: 10.0, release: 100.0, master_level: 0.8
